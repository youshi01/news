# 部署说明

## 默认后台入口

```text
后台地址：/manage-8f3k2
登录地址：/manage-8f3k2/login
默认账号：admin
默认密码：ChangeMe_2026_admin
默认安装口令：ChangeMe_Install_2026
```

`/admin` 会返回 404，`robots.txt` 不会暴露后台地址，前台页面也不会显示后台入口。

上线后建议立刻修改：

```env
ADMIN_PATH=/your-private-admin-path
ADMIN_USER=your_admin_name
ADMIN_PASSWORD=your_long_random_password
ADMIN_INTERNAL_TOKEN=your_long_internal_token
ADMIN_COOKIE_SECURE=false
INSTALL_TOKEN=your_long_install_token
ENABLE_RSS_IMPORT=true
```

本地或直接访问 `http://服务器IP:3000` 时，`ADMIN_COOKIE_SECURE` 必须保持 `false`。如果这里设成 `true`，浏览器会因为不是 HTTPS 而不保存登录 Cookie，表现就是登录后点后台菜单又跳回登录页。正式 HTTPS 反代后可以改成 `true`。

## 启动镜像

```bash
docker pull ghcr.io/youshi01/news:latest
docker run -d \
  --name news \
  -p 3000:3000 \
  --add-host=host.docker.internal:host-gateway \
  -v $(pwd)/data:/app/data \
  -e ADMIN_PATH=/manage-8f3k2 \
  -e ADMIN_USER=admin \
  -e ADMIN_PASSWORD=ChangeMe_2026_admin \
  -e ADMIN_INTERNAL_TOKEN=ChangeMe_Internal_Admin_2026 \
  -e ADMIN_COOKIE_SECURE=false \
  -e INSTALL_TOKEN=ChangeMe_Install_2026 \
  -e ENABLE_RSS_IMPORT=true \
  ghcr.io/youshi01/news:latest
```

首次部署后先打开主页：

```text
http://服务器IP:3000/
```

如果还没有配置数据库，系统会自动进入初始化页面：

```text
/install
```

填写 MySQL 连接信息后，系统会自动创建数据库表，并把运行时配置保存到：

```text
data/install.json
```

如果 MySQL 在宿主机上，MySQL 主机优先填写：

```text
host.docker.internal
```

Linux 服务器如果无法解析这个地址，请填写宿主机内网 IP，或者运行容器时保留：

```bash
--add-host=host.docker.internal:host-gateway
```

## 文章采集

后台可手动触发：

```text
来源管理：/manage-8f3k2/sources
点击：立即导入 RSS 文章
```

Docker 镜像部署时，命令需要在容器里执行：

```bash
docker exec news npm run import:feeds
```

不要在服务器 `/home` 目录直接运行 `npm run import:feeds`，那里没有项目的 `package.json`。

热点词导入：

```bash
docker exec news npm run import:hot-news
```

如果使用 `docker compose`：

```bash
docker compose exec web npm run import:feeds
docker compose exec web npm run import:hot-news
```

worker 会等待数据库初始化完成，然后按照 `FETCH_INTERVAL_MINUTES` 自动采集。RSS 自动采集默认开启：

```env
ENABLE_RSS_IMPORT=true
MAX_ITEMS_PER_SOURCE=12
FETCH_INTERVAL_MINUTES=30
```

## 检查服务

如果 `/install` 显示 `{"code":404,"msg":"Not Found"}`，先访问：

```text
http://服务器IP:3000/api/health
```

正常应该返回 `app: "sea-news-hub"` 和 `image: "ghcr.io/youshi01/news"`。如果不是，说明当前 3000 端口不是这个新闻站容器在响应。
