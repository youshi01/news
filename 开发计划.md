# 自研动态聚合站方案：Next.js/Nuxt + PostgreSQL + Meilisearch

## 1. 项目定位

目标做一个面向海外流量的动态聚合站：

```text
新闻聚合 + AI 摘要 + 分类/标签 + 全文搜索 + SEO 页面 + 多来源自动导入
```

核心体验：

```text
把文章文件 / RSS / URL 导入系统
→ 自动解析
→ 入库
→ Meilisearch 建索引
→ 前台可搜索
→ 文章页可被 Google 收录
```

适合方向：

```text
Tech News
AI News
Cybersecurity News
Crypto News
Startup News
SaaS News
Proxy / VPN / Hosting News
```

## 2. 推荐技术栈

### 前端 / 服务端

推荐二选一：

```text
Next.js 15 + App Router
Nuxt 3
```

如果主要做英文 SEO，我更建议：

```text
Next.js
```

原因：生态更大，部署资料多，SEO/SSR/ISR 支持成熟。

### 数据库

```text
PostgreSQL
```

原因：稳定、适合文章/标签/来源/任务记录。

### 搜索

```text
Meilisearch
```

用途：全文搜索、标签筛选、来源筛选、时间排序、热度排序。

### 队列 / 定时任务

简单版：

```text
Node Cron
```

进阶版：

```text
BullMQ + Redis
```

### 内容抓取

```text
RSS Parser
Readability
Cheerio
Playwright 可选
```

### AI 摘要

```text
OpenAI / Claude / Gemini / 中转 API
```

### 部署

推荐：

```text
Docker Compose
Nginx / Caddy
Cloudflare
```

## 6. 内容导入流程

### 6.1 RSS 自动导入

流程：

```text
读取 sources 表
→ 抓 RSS
→ 获取 item 标题/链接/发布时间
→ 去重
→ 抓正文
→ Readability 提取正文
→ AI 摘要/改写
→ 入库为 draft 或 published
→ 同步到 Meilisearch
```

### 6.2 文件导入

支持把 Markdown / JSON 文件放进目录：

```text
imports/articles/*.md
imports/articles/*.json
```

执行：

```bash
npm run import:files
```

自动：

```text
读取文件
→ 解析 frontmatter
→ 入库
→ 建索引
```

### 6.3 URL 手动导入

后台提供：

```text
粘贴 URL
→ 抓取网页
→ 提取正文
→ AI 摘要
→ 编辑确认
→ 发布
```

## 9. 反采集/合规策略

不要全文复制来源站。

推荐文章结构：

```text
标题：自写
摘要：AI/人工改写
正文：
  - What happened
  - Why it matters
  - Key details
  - Impact
  - Our take
  - Source links
```

每篇保留来源链接：

```text
Source: example.com
```

不建议：

```text
全文搬运
复制图片
复制标题不改
复制品牌风格
无来源引用
```

## 11. Docker Compose 服务

建议服务：

```text
web          Next.js
postgres     PostgreSQL
meilisearch  搜索服务
redis         可选，任务队列
worker        抓取/索引任务
nginx         反代
```

示例：

```yaml
services:
  postgres:
    image: postgres:16
    environment:
      POSTGRES_DB: news
      POSTGRES_USER: news
      POSTGRES_PASSWORD: change_me
    volumes:
      - postgres_data:/var/lib/postgresql/data

  meilisearch:
    image: getmeili/meilisearch:v1.8
    environment:
      MEILI_MASTER_KEY: change_me
    volumes:
      - meili_data:/meili_data

  web:
    build: .
    env_file: .env
    depends_on:
      - postgres
      - meilisearch

volumes:
  postgres_data:
  meili_data:
```

## 15. 最终建议

方案 B 适合长期做，因为它比 WordPress 更可控：

```text
可以自动抓取
可以自动摘要
可以全文搜索
可以程序化 SEO
可以做多语言
可以做 API
可以做多个垂直站点
```

但 MVP 不要做太大。

第一版目标：

```text
能导入 RSS
能生成文章
能搜索
能被 Google 收录
能每天自动更新
```

先跑起来，再做多语言、热点榜、推荐系统和 API。
