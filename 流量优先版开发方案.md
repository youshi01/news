# 东南亚多语言热点新闻站：流量优先版开发方案

## 1. 项目目标

本项目优先目标不是先做复杂后台，而是先把站点快速部署上线，持续产出可被搜索引擎收录的多语言热点新闻页面，尽快验证自然流量。

核心目标：

```text
热点抓取
→ 自动生成新闻页面
→ 自动生成 SEO 元数据
→ 自动更新 sitemap / RSS
→ 后台查看流量、停留时长、点击次数
→ 流量起来后接广告
```

第一阶段只追求：

```text
能快速部署
能稳定抓取热点
能自动生成页面
能自动做基础 SEO
能看到每篇文章的数据
能预留广告位
```

## 2. 目标市场和语言

面向东南亚流量，建议第一阶段不要一次做太多语言，先做 3-4 个重点语言。

MVP 推荐：

```text
en      英文，默认语言和兜底语言
id      印尼语，东南亚优先市场
vi      越南语，搜索需求强
th      泰语，可作为第三重点语言
```

第二阶段可增加：

```text
ms-MY   马来语
tl-PH   菲律宾 / Tagalog
zh-Hans 马来西亚华语内容，可后续视流量决定
```

URL 使用子目录，不使用 query 参数：

```text
/en/news/openai-launches-new-model
/id/news/openai-luncurkan-model-baru
/vi/news/openai-ra-mat-mo-hinh-moi
/th/news/...
```

这样更利于搜索引擎理解不同语言版本，也方便后续做 hreflang 和多语言 sitemap。

## 3. 技术栈

推荐使用：

```text
Next.js 当前稳定版 + App Router
MySQL
Prisma 或 Drizzle
Docker
Nginx / Caddy
Cloudflare
Meilisearch，可第二阶段接入
Node Cron / BullMQ，可从简单定时任务开始
```

调整点：

```text
数据库使用你服务器已有的 MySQL
Docker 只打包应用，不在 Docker Compose 里启动 MySQL
部署时通过 DATABASE_URL 连接外部 MySQL
初始化使用 init.sql 或迁移脚本执行
```

## 4. 部署方式

Docker 只负责应用服务和可选 worker。

推荐服务结构：

```text
web       Next.js 前台 + 后台
worker    热点抓取 / 翻译 / SEO 生成 / sitemap 刷新任务
nginx     反向代理，可选
```

不建议第一阶段在 Docker 里打包 MySQL，因为你服务器已经有 MySQL。

示例环境变量：

```env
DATABASE_URL="mysql://user:password@mysql-host:3306/news_site"
SITE_URL="https://example.com"
DEFAULT_LOCALE="en"
SUPPORTED_LOCALES="en,id,vi,th"
CRON_SECRET="change_me"
```

示例 Docker Compose：

```yaml
services:
  web:
    build: .
    restart: always
    env_file: .env
    ports:
      - "3000:3000"

  worker:
    build: .
    restart: always
    env_file: .env
    command: npm run worker
```

MySQL 初始化：

```text
1. 在服务器 MySQL 创建数据库 news_site
2. 导入 sql/init.sql
3. 配置 .env 的 DATABASE_URL
4. docker compose up -d
5. 后台添加热点源或直接运行抓取任务
```

## 5. 核心业务流程

热点新闻流水线：

```text
抓取热点源 / RSS / 热榜
→ 去重
→ 提取标题、链接、发布时间、摘要
→ 抓取正文或生成短摘要
→ 生成英文基础版本
→ 翻译/本地化到目标语言
→ 生成 slug、SEO title、SEO description
→ 入库
→ 生成页面
→ 更新 sitemap
→ 后台开始统计流量
```

第一阶段可以不做全文搬运，优先做：

```text
标题
简短摘要
What happened
Why it matters
关键点
来源链接
```

这样更适合热点站，也降低版权风险。

## 6. 自动 SEO 能否实现

可以实现“添加一篇新闻后，站内 SEO 立即生成”。

能立即生成的内容：

```text
独立文章 URL
title
meta description
canonical
OpenGraph
Twitter Card
NewsArticle Schema
Breadcrumb Schema
多语言 hreflang
sitemap 记录
RSS feed 记录
分类页 / 标签页聚合
```

不能保证的内容：

```text
Google 立刻收录
Google 立刻排名
Google 立刻展示在新闻结果
```

原因是搜索引擎收录取决于抓取频率、站点权重、内容质量、内链结构、重复度和来源可信度。

因此正确目标是：

```text
新闻发布后，站内 SEO 立即准备好
sitemap 和 RSS 立即更新
首页 / 分类页 / 最新页立即出现内链
等待搜索引擎抓取
```

## 8. 后台流量统计

后台需要能看：

```text
PV 页面浏览次数
UV 独立访客数
文章浏览次数
来源渠道
搜索关键词，可后续通过 Search Console 补充
平均浏览时长
滚动深度
点击次数
出站来源点击
语言维度数据
国家/地区维度数据
设备类型
热门文章
热门分类
热门标签
```

第一阶段建议自己记录核心事件：

```text
page_view
article_view
click
outbound_click
ad_slot_view_reserved
scroll_depth
heartbeat
```

浏览时间统计方式：

```text
进入文章页时记录 article_view
页面每 15 秒发送一次 heartbeat
用户离开页面时记录最后一次停留时间
后台按 session 聚合阅读时长
```

注意：

```text
统计尽量轻量
不要影响页面速度
不要阻塞 SEO 页面渲染
可以先用数据库记录，流量大后再迁移到 ClickHouse / Umami / Plausible / PostHog
```

## 9. 广告位预留

第一阶段不展示广告，但代码和数据库要预留广告位。

预留位置：

```text
首页信息流顶部
首页信息流中部
文章页标题下
文章页正文中段
文章页正文底部
侧边栏，桌面端
分类页列表中部
移动端 sticky bottom，后续谨慎开启
```

广告位只做占位配置，不在页面展示广告内容。

数据表保留：

```text
ad_slots
ad_events
```

前端组件保留：

```text
<AdSlot placement="article_after_title" hiddenUntilEnabled />
```

当流量起来后，只需要：

```text
后台开启广告位
填入广告代码或广告平台 ID
前台开始展示
统计曝光和点击
```

## 10. 数据库设计

使用 MySQL。

核心表：

```text
articles                  新闻主表
article_translations      多语言内容表
sources                   来源表
categories                分类表
category_translations     分类翻译表
tags                      标签表
tag_translations          标签翻译表
article_tags              新闻标签关联表
traffic_events            流量事件表
daily_article_stats       每日文章统计表
click_events              点击事件表
ad_slots                  广告位表
ad_events                 广告事件表，预留
import_tasks              抓取任务表
site_settings             站点配置表
```

建议 `articles` 存放与语言无关的信息：

```text
id
source_id
source_url
canonical_url
url_hash
content_hash
original_language
published_at
imported_at
status
cluster_key
created_at
updated_at
```

`article_translations` 存放每种语言的 SEO 和内容：

```text
id
article_id
locale
slug
title
description
summary
content_html
seo_title
seo_description
og_image
translation_status
review_status
created_at
updated_at
```

重要唯一约束：

```text
articles.url_hash UNIQUE
article_translations(locale, slug) UNIQUE
article_translations(article_id, locale) UNIQUE
```

## 11. 初始化 SQL 草案

正式开发时将下面内容整理为：

```text
sql/init.sql
```

初始化 SQL 示例：

```sql
CREATE TABLE sources (
  id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
  name VARCHAR(190) NOT NULL,
  site_url VARCHAR(500) NOT NULL,
  rss_url VARCHAR(500) NULL,
  source_type VARCHAR(50) NOT NULL DEFAULT 'rss',
  default_locale VARCHAR(20) NOT NULL DEFAULT 'en',
  enabled TINYINT(1) NOT NULL DEFAULT 1,
  last_fetched_at DATETIME NULL,
  fetch_interval_minutes INT NOT NULL DEFAULT 60,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

CREATE TABLE articles (
  id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
  source_id BIGINT UNSIGNED NULL,
  source_url VARCHAR(700) NOT NULL,
  canonical_url VARCHAR(700) NULL,
  url_hash CHAR(64) NOT NULL,
  content_hash CHAR(64) NULL,
  original_language VARCHAR(20) NOT NULL DEFAULT 'en',
  published_at DATETIME NULL,
  imported_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  status VARCHAR(30) NOT NULL DEFAULT 'published',
  cluster_key VARCHAR(190) NULL,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  UNIQUE KEY uniq_articles_url_hash (url_hash),
  KEY idx_articles_published_at (published_at),
  KEY idx_articles_status (status),
  KEY idx_articles_source_id (source_id)
);

CREATE TABLE article_translations (
  id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
  article_id BIGINT UNSIGNED NOT NULL,
  locale VARCHAR(20) NOT NULL,
  slug VARCHAR(220) NOT NULL,
  title VARCHAR(300) NOT NULL,
  description VARCHAR(500) NULL,
  summary TEXT NULL,
  content_html MEDIUMTEXT NULL,
  seo_title VARCHAR(300) NULL,
  seo_description VARCHAR(500) NULL,
  og_image VARCHAR(700) NULL,
  translation_status VARCHAR(30) NOT NULL DEFAULT 'done',
  review_status VARCHAR(30) NOT NULL DEFAULT 'auto',
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  UNIQUE KEY uniq_translation_locale_slug (locale, slug),
  UNIQUE KEY uniq_translation_article_locale (article_id, locale),
  KEY idx_translation_locale (locale),
  KEY idx_translation_article_id (article_id)
);

CREATE TABLE categories (
  id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
  slug VARCHAR(190) NOT NULL,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  UNIQUE KEY uniq_categories_slug (slug)
);

CREATE TABLE category_translations (
  id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
  category_id BIGINT UNSIGNED NOT NULL,
  locale VARCHAR(20) NOT NULL,
  name VARCHAR(190) NOT NULL,
  description VARCHAR(500) NULL,
  UNIQUE KEY uniq_category_locale (category_id, locale)
);

CREATE TABLE tags (
  id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
  slug VARCHAR(190) NOT NULL,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  UNIQUE KEY uniq_tags_slug (slug)
);

CREATE TABLE tag_translations (
  id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
  tag_id BIGINT UNSIGNED NOT NULL,
  locale VARCHAR(20) NOT NULL,
  name VARCHAR(190) NOT NULL,
  UNIQUE KEY uniq_tag_locale (tag_id, locale)
);

CREATE TABLE article_tags (
  article_id BIGINT UNSIGNED NOT NULL,
  tag_id BIGINT UNSIGNED NOT NULL,
  PRIMARY KEY (article_id, tag_id)
);

CREATE TABLE traffic_events (
  id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
  event_type VARCHAR(50) NOT NULL,
  article_id BIGINT UNSIGNED NULL,

  path VARCHAR(700) NOT NULL,
  referrer VARCHAR(700) NULL,
  session_id VARCHAR(100) NULL,
  visitor_id VARCHAR(100) NULL,
  country VARCHAR(80) NULL,
  device_type VARCHAR(50) NULL,
  duration_seconds INT NULL,
  scroll_depth TINYINT UNSIGNED NULL,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  KEY idx_traffic_created_at (created_at),
  KEY idx_traffic_article_id (article_id),
  KEY idx_traffic_event_type (event_type),
  KEY idx_traffic_locale (locale)
);

CREATE TABLE click_events (
  id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,

  target_type VARCHAR(50) NULL,
  session_id VARCHAR(100) NULL,
  visitor_id VARCHAR(100) NULL,

CREATE TABLE daily_article_stats (

  page_views INT UNSIGNED NOT NULL DEFAULT 0,

  avg_duration_seconds INT UNSIGNED NOT NULL DEFAULT 0,

  updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,

CREATE TABLE ad_slots (
  id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,

CREATE TABLE ad_events (

CREATE TABLE import_tasks (

  count_created INT UNSIGNED NOT NULL DEFAULT 0,

CREATE TABLE site_settings (
```

## 13. 自动化 SEO 规则

每篇新闻发布时自动生成：

```text
slug
seo_title
seo_description
canonical
hreflang
OpenGraph
Twitter Card
NewsArticle Schema
Breadcrumb Schema
内链推荐
sitemap 记录
RSS feed 记录
```

标题规则：

```text
保留核心关键词
避免标题党
每种语言单独生成自然标题
长度控制在搜索结果可读范围内
```

描述规则：

```text
说明发生了什么
说明为什么重要
包含主要实体词
不同语言不要完全机械直译
```

结构化数据：

```text
NewsArticle
headline
description
datePublished
dateModified
author / publisher
image，可选
mainEntityOfPage
```

sitemap 策略：

```text
普通 sitemap 包含所有可收录页面
news sitemap 只包含近期新闻
每次新增或更新新闻后自动更新 lastmod
多语言版本用 alternates / hreflang 关联
```

## 14. 合规和版权

不建议全文搬运来源站内容。

建议内容结构：

```text
自写标题
AI 辅助摘要
关键点整理
背景解释
影响分析
来源链接
```

不要做：

```text
直接复制全文
复制原站图片
复制标题不改
隐藏来源
绕过付费墙
无节制高频抓取
```

需要预留：

```text
来源引用
DMCA / 移除请求联系入口
编辑政策
隐私政策
robots 和来源抓取规则检查
```

```text

4. 初始化 SQL
```

```text

```

## 17. 关键结论

你的思路可以实现，但要分清两件事：

```text
站内 SEO 自动化：可以做到新增新闻后立即生成
搜索引擎收录和排名：不能保证立即发生
```

所以第一版的正确目标是：

```text
快速部署
快速产出可收录页面
快速统计流量
快速验证哪些语言和新闻类型有流量
等流量起来后直接开启广告位
```
